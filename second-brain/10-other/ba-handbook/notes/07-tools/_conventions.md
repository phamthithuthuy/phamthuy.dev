# Conventions — Tools

> File này **mở rộng** [[note-guidelines|Note guidelines]] cho riêng nhóm `07-tools`. Chỉ thêm hoặc siết chặt luật chung. Nếu mâu thuẫn luật chung, **luật chung thắng**.

## Phạm vi

Áp dụng cho note về công cụ và literacy kỹ thuật cho BA: Jira, Confluence, Figma, draw.io, SQL, security awareness.

## Hard rules

| Rule | Vì sao |
|---|---|
| Note tool tập trung vào **thao tác phục vụ công việc BA**, không phải hướng dẫn dùng tool toàn diện. | BA cần đủ dùng, không cần thành chuyên gia tool; tránh trùng tài liệu chính thức. |
| Nếu nội dung **phụ thuộc phiên bản** (UI, cú pháp), ghi rõ **version/ngày** kiểm chứng. | UI tool đổi liên tục; note không ghi version sẽ lạc hậu âm thầm. |
| Ví dụ SQL/test case phải **chạy được hoặc nêu rõ schema giả định**. | Snippet không chạy được làm người đọc mất thời gian sửa. |

## Default conventions

- Ưu tiên link tới tài liệu chính thức của tool cho phần chi tiết, note chỉ giữ phần "BA dùng thế nào".
- SQL ví dụ mặc định dùng cú pháp ANSI; nếu dùng dialect riêng (MySQL, Postgres) thì ghi rõ.

## Local vocabulary

Chưa có thuật ngữ cục bộ riêng; thuật ngữ kỹ thuật dùng chung [[../glossary|Glossary]] khi liên quan BA.

## Khi nào KHÔNG áp dụng

- Note khái niệm kỹ thuật không gắn version (ví dụ "SQL JOIN là gì") không cần ghi version.
